Evacuating a Ganeti node for hardware diagnostics

Sat 23 March 2013
By William Van Hevelingen

In ganeti.

tags: ganetikvm

Log into the master ganeti node.

Run the gnt-node migrate command, passing in the node to migrate off of

sudo gnt-node migrate node2.cat.pdx.edu

The output should look similar to this:

After the primaries have been migrated run the gnt-node evacuate command to move the secondaries

sudo gnt-node evacuate --secondary-only katana.cat.pdx.edu

If the evacauate command fails you can move secondaries per instance

gnt-instance replace-disks -n node1.cat.pdx.edu myinstance.cat.pdx.edu
